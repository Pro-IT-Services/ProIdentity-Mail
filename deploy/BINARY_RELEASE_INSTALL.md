# Binary Release Install

Use this path when the target server should install ProIdentity Mail from a
published binary release without compiling Go code on the server.

`install-from-release.sh` is the release downloader and launcher. After it
downloads and extracts the archive, it runs `proidentity-production-setup.sh`;
that bootstrap script installs packages, creates users, configures MariaDB,
Postfix, Dovecot, Rspamd, ClamAV, Redis, Unbound, Nginx, writes systemd units,
starts services, and generates missing secrets.

## Release Archives

GitHub Actions and GitLab CI create these archives when a version tag is pushed:

- `proidentity-mail_<version>_linux_x64.tar.gz`
- `proidentity-mail_<version>_linux_x86.tar.gz`
- `proidentity-mail_<version>_linux_arm.tar.gz`
- `proidentity-mail_<version>_linux_arm64.tar.gz`
- `SHA256SUMS`

Each archive contains the binaries and setup scripts expected by
`proidentity-production-setup.sh`.

## Install

From GitHub releases:

```bash
curl -fsSL https://github.com/Pro-IT-Services/ProIdentity-Mail/raw/main/deploy/install-from-release.sh \
  -o /tmp/install-proidentity-mail.sh
chmod +x /tmp/install-proidentity-mail.sh

sudo /tmp/install-proidentity-mail.sh \
  --github-repo Pro-IT-Services/ProIdentity-Mail \
  --version v0.2.0 \
  -- \
  --public-ipv4 203.0.113.10 \
  --mail-hostname mail.example.com \
  --admin-hostname madmin.example.com \
  --webmail-hostname webmail.example.com \
  --dav-hostname webmail.example.com \
  --autoconfig-hostname autoconfig.example.com \
  --autodiscover-hostname autodiscover.example.com \
  --tls-mode letsencrypt-dns-cloudflare
```

From GitLab Generic Package Registry:

```bash
curl -fsSL https://gitlab.com/YOUR_NAMESPACE/proidentity-mail/-/raw/main/deploy/install-from-release.sh \
  -o /tmp/install-proidentity-mail.sh
chmod +x /tmp/install-proidentity-mail.sh

sudo /tmp/install-proidentity-mail.sh \
  --gitlab-project-id YOUR_GITLAB_PROJECT_ID \
  --version v0.2.0 \
  -- \
  --public-ipv4 203.0.113.10 \
  --mail-hostname mail.example.com \
  --admin-hostname madmin.example.com \
  --webmail-hostname webmail.example.com \
  --dav-hostname webmail.example.com \
  --autoconfig-hostname autoconfig.example.com \
  --autodiscover-hostname autodiscover.example.com \
  --tls-mode letsencrypt-dns-cloudflare
```

## Direct URL Install

```bash
sudo /tmp/install-proidentity-mail.sh \
  --release-url https://example.com/proidentity-mail_v0.2.0_linux_x64.tar.gz \
  --checksum-url https://example.com/SHA256SUMS \
  -- \
  --mail-hostname mail.example.com \
  --admin-hostname madmin.example.com \
  --webmail-hostname webmail.example.com
```

## Notes

- The target server does not need Go installed.
- Missing secrets are generated by `proidentity-production-setup.sh`.
- The full install summary is written to `/root/proidentity-install-summary.txt`
  unless `--summary-file` is passed to the setup script.
- Supported TLS modes are `none`, `behind-proxy`, `letsencrypt-http`,
  `letsencrypt-dns-cloudflare`, and `custom-cert`.
- Use `letsencrypt-http` when public DNS points directly to this server and
  port 80 reaches local Nginx. Use `letsencrypt-dns-cloudflare` when Cloudflare
  proxy or another upstream proxy prevents HTTP-01 from reaching the origin.
